# urb-research

> MCP-сервер реестра исследований: превращает тему в структурированный, снабжённый ссылками разбор на основе отобранных веб-источников. Долговременная память ресёрча — всё созданное видно в веб-интерфейсе, это и есть результат работы, а не черновик.

Приложение: Vue 3 web-интерфейс + FastAPI-бекенд + база данных (PostgreSQL или SQLite). Единая точка входа — `src/app.py`. Собранный веб-интерфейс лежит прямо в репозитории, и бекенд раздаёт его сам, поэтому для запуска не нужны ни Node.js, ни nginx.

Ключевые модули: **research** — реестр исследований и MCP-сервер поверх него; **web_search** — веб-поиск и сохранение страниц; **core_connectors** — коннекторы к внешним API (Tavily / Firecrawl / xAI / OpenRouter / OpenAI / Anthropic).

## Требования

Для запуска нужны всего два инструмента:

- **git** — забрать код.
- **uv** — менеджер Python-проектов от Astral. Он сам скачивает нужный Python (проект требует 3.12, версия зафиксирована в `.python-version`), поэтому отдельно ставить Python не нужно.

Базу данных ставить не нужно: по умолчанию используется **SQLite** — один файл, без отдельного сервера. PostgreSQL — опция для боевого масштаба (потребуется сервер 14+ с созданными базой и пользователем).

## Установка на чистую систему

**1. Установите git и uv.**

```bash
# git — если ещё не стоит (Debian/Ubuntu)
sudo apt-get update && sudo apt-get install -y git curl
```

```bash
# uv — официальный установщик (ставит в ~/.local/bin)
curl -LsSf https://astral.sh/uv/install.sh | sh
# подхватить PATH в текущей сессии (или просто перелогиньтесь)
source "$HOME/.local/bin/env"
uv --version
```

**2. Заберите код и установите зависимости.**

```bash
git clone <repo-url> urb-research && cd urb-research
# Python-зависимости (uv сам скачает Python 3.12 при первом запуске)
uv sync
```

> Python-пакеты ставятся готовыми бинарными сборками (pillow, lxml, asyncpg, pikepdf и др.) — компилятор и системные dev-библиотеки не нужны.

> **Файл `.env` не обязателен.** Из коробки приложение работает на встроенных значениях: режим `prod`, база — SQLite в `runtime/prod/app.sqlite3`, схема БД накатывается сама при первом запуске. Свой `.env` нужен, только если хотите сменить порт, хост или перейти на PostgreSQL — за образец возьмите `.env.example.prod`.

> **Ключи внешних сервисов** (Tavily / Firecrawl / xAI / OpenRouter / OpenAI / Anthropic) задаются не в `.env`, а прямо в веб-интерфейсе на странице «Настройки» после запуска. Для проверки интерфейса они не нужны.

## Запуск

```bash
uv run python src/app.py --backend --worker
```

Бекенд поднимается на порту `13410` (по умолчанию) и сам раздаёт веб-интерфейс. Откройте в браузере:

```
http://localhost:13410
```

Флаги `--backend` (HTTP-сервер) и `--worker` (фоновые задачи) можно разнести по разным процессам, но для одиночного запуска достаточно указать оба.

## Подключение MCP

Держать сервер запущенным вручную не нужно: MCP-клиент подключается через **stdio-шим** (`src/app.py --mcp-stdio`). Клиент сам запускает процесс при открытии сессии — тот поднимает бекенд, открывает браузер на главной и передаёт вызовы инструментов на живой сервер.

В конфиге MCP-клиента (Claude Desktop / Claude Code) добавьте запись с `command` (не `url`):

```json
{
  "mcpServers": {
    "urb-research": {
      "command": "/path/to/uv",
      "args": [
        "run", "--directory", "/path/to/urb-research",
        "python", "src/app.py", "--mcp-stdio"
      ]
    }
  }
}
```

> Укажите **абсолютный** путь к `uv` и к проекту (`--directory`). Многие MCP-клиенты стартуют из своего каталога с урезанным PATH, и без `--directory` uv не найдёт проект → «Connection Failed».

Подробности шима и его настройки (`MCP_STDIO_*` в `.env`) — [MCP stdio-шим](AGENTS/docs/platform/mcp-stdio-shim.md).

## Структура

```
├── src/        # код: app.py (точка входа), core/ (платформа), modules/ (домены)
├── web/        # веб-интерфейс на Vue 3; web/dist/ — собранный SPA (коммитится в git)
├── storage/    # файловое хранилище
├── runtime/    # данные, создаваемые при работе: БД (SQLite), кеш, логи
├── tests/      # тесты
└── AGENTS/     # материалы для AI-агентов и внутренняя документация
```
