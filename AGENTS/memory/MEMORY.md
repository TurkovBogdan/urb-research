## 1. Always-apply rules (obey every session)

- 🔴 [**NEVER destructive DDL in ad-hoc scripts**](standalone_script_hits_dev_db.md) — a plain `uv run python`/`asyncpg` script uses `Config()` → the **DEV DB** (`APP_ENV=test` does NOT redirect it; only pytest's conftest does, to in-memory sqlite). NEVER run `DROP SCHEMA`/`DROP TABLE`/`TRUNCATE`/`DELETE` outside pytest. Touch a real Postgres DB only by explicit literal name + `assert current_database()`. **This wiped the whole dev DB once (2026-06-14).**
- **Discussion ≠ go** — never edit code (`src/`/`web/`/tests/configs) while a task is still being discussed; mutate code only on an explicit go-ahead (записи — tasks/memory/docs — exempt).
- **Never guess, verify** — never guess file names/paths/symbols/columns; confirm with Glob/Grep/Read/dev-query before acting.
- **Re-read before edit** — Read a doc/file immediately before editing it; the user edits in parallel, cached state goes stale.
- **Edit by hand, no scripts** — never sed/scripted rewrites for code edits (even bulk renames); Edit each file by hand; Bash only for read-only discovery + tests.
- **Move files with mv** — `mv` to relocate, then Edit only the changed lines; never rewrite a whole file just to move it.
- **Self-documenting code** — code explains itself via structure + names, NOT comments; unclear logic → extract & name it (variable/predicate/function). No label comments (`# Tier 1 …`) or prose in code; comments justify the *why*, never the *what*. [conventions/code-style](../docs/conventions/code-style.md).
- **Comment before command** — in code blocks, comment on its own line above the command; no trailing inline comments.
- [Commit protocol](feedback_commit_protocol.md) — HARD: never commit yourself; stage only own changes with **one `git add <path>` per file** (never `-A`/`-a`), then a single one-line English `git commit -m` (no body/co-authors); flag shared INDEX files.
- **Commit language English** — commit messages ALWAYS in English (chat/discussion stays Russian).
- [Filters in VCard](feedback_filters_in_vcard.md) — filter bars on list pages ALWAYS wrapped in a VCard (`variant="outlined" rounded="lg" class="filter-panel mb-3"`, pad 12px); never bare.
- [No-repeat caveat](feedback_no_repeat_caveat.md) — say a caveat/prerequisite once; never re-raise it in later turns of the same thread (nagging).
- [Browser isolation](feedback_browser_isolation.md) — NEVER drive the user's working browser; `list_connected_browsers` → `select_browser` a dedicated instance before any page interaction.
- [Standalone script hits dev DB](standalone_script_hits_dev_db.md) — ad-hoc `uv run python` + `Config()` connects to **dev** (the in-memory-sqlite redirect lives only in pytest conftest); NEVER run DROP SCHEMA/TRUNCATE in such scripts; target real Postgres DBs by explicit name + assert `current_database()`.
- [Edge cases into tests](feedback_edge_cases_into_tests.md) — every strange/edge case found (defects, accepted limitations, deferred cases) → fixture + test; expand the test set after EACH fix iteration; what a fixture can't reproduce (e.g. CRLF) → a unit test.
- 🔴 [Never checkout dirty shared files](feedback_never_checkout_dirty_shared_files.md) — NEVER `git checkout`/`restore`/`stash` a tracked file to undo your own edit; the working tree holds OTHER tasks' uncommitted work (esp. `web/src/locales/**`, shared INDEX files) → unrecoverable. Undo by hand (Edit). Wiped uncommitted i18n once (2026-06-17).
- [No backtick wrapping](feedback_no_backtick_wrapping.md) — don't wrap terms/identifiers in inline-code backticks in Russian chat replies; plain text. Code blocks still fine.
- [Blockquote explanations](feedback_blockquote_explanations.md) — пояснения (определения, оговорки, «почему») оформляй цитатой `>` — в чате и в телах заметок urb-research; шаги/чек-листы обычным текстом.

Тройка `web_search_query`→`web_search_query_result`→`web_search_page`. **PK — строковые читаемые коды** `String(25)` (3-символьный префикс + 22 hex, чтобы нейронка отличала прогон от документа): query `code=WQ_<random_hash()>` (случайный — дедупа у прогона нет), page `code=WD_<text_hash(url)>` (детерминированный = сам код и есть ключ дедупа по url); числового `id` на query/page **больше нет** (`hashing.random_hash()` = 22-символьный случайный hex, тот же алфавит/длина, что у `text_hash`). `web_search_query_result` (результаты запроса) оставляет суррогатный int `id`, FK — `query_code`/`page_code` (`String(25)`), unique `(query_code,page_code)`. (portable `json_value`; норм. url = всегда https; миграции `wsm_001..003` на портируемых типах → идут и на sqlite, и на PG (core `com_*` и research `rem_*` тоже); **dev-sqlite строится миграциями, не create_all**). ORM `WebSearchQuery`/`WebSearchQueryResult`/`WebSearchPage` (файлы `models/{query,page,query_result}.py`) — **каждый держит ORM + свой read-DTO рядом** (`QueryRow`/`QueryDetail`, `PageRow`/`PageDetail`, `QueryResultView`; `dto.py` нет); `Paged` и даты `DatetimeUTCStr` — ядровые (`core.api.Paged`/`core.utils.date`, дедуп: research тоже на них); колонки `query` (query-таблица), `query_code`/`page_code` (query_result). Cross-module soft-ref: `research_query.search_code:String(25)`(→`WQ_`), `research_document.page_code:String(25)`(→`WD_`). **Две независимые роли провайдера**, у каждой своя настройка по умолчанию: движок поиска `SearchEngine.search(req)->[{url,rank?,score?,meta?}]` (**только ссылки**) и движок контента `FetchEngine.fetch_pages(urls)->{url:md}`+`pages_per_request`. Tavily/Firecrawl умеют обе → регистрируются в обоих реестрах; **Grok (xai) — только поиск** (нет search-эндпойнта: поиск = **агентский** `core_connectors` xAI `responses()` + server-side `web_search` tool + **строгий structured output** — модель возвращает JSON `links[]` url/title/reason/relevance по `AGENT_INSTRUCTION`; URL печатает сама → **grounding-фильтр** оставляет лишь встреченные в `web_search_call.action.sources`, `relevance→score`/`meta={title,reason}`; эталон — бенч `verify_relevant_links.py`). Query снимает **оба** движка в колонки `search_engine`+`fetch_engine` (`String(64)`); ошибка — колонка `error` (nullable); `max_results` и прочие параметры запроса — в `params` (отдельной колонки max_results нет). Активные движки — настройки `search_engine`/`fetch_engine` (термин **engine**, не provider — по всей цепочке). **Исполнение синхронное** — ни очереди, ни планировщик-задач, ни ретраев (`module.py` без `configure()`/`register_tasks`). `services/searcher.Searcher` — **статический фасад**: публичные `search_engines()`/`fetch_engines()` (коды доступных движков = `available_codes()`) + два запуска: `search(...)` (**блокирующий**, вернуть завершённый — MCP/тесты) и `submit(...)` (**fire-and-forget**: создать `pending`, прогон фоном `asyncio.create_task`, вернуть строку сразу — HTTP-создание; оба делят `_prepare`/`_run`), сигнатура `(query, *, search_engine=None, fetch_engine=None, max_results=None→default_pages, …)` (движки опциональны, None → приватные `_default_search_engine()`/`_default_fetch_engine()` = настройки). За один `await`: резолвит движки (`<registry>.get(<param> or <default>)`) → `query_create` (строка `pending`, снимок `search_engine`+`fetch_engine`; `to_params()` кладёт в `params` все параметры запроса, включая `max_results`) → `query_mark_processing` (`processing`) → `engine.search(request)` (**ссылки, без контента — инлайн-контент больше не тянем**) → `_store_results` (приватный шаг searcher, не отдельный сервис — `save.py` удалён; создаёт `pending`-страницы) → `_fetch_pages(fetcher, …)` (батчами по `fetcher.pages_per_request`: `pages_mark_processing`→фетч→`done`/`error`; параллельно под семафором `fetch_concurrency`) → `query_finish` (`done`, даже без результатов) → возвращает завершённый `WebSearchQuery`. Статусы — **два набора по ролям** (`constants.py`), одинаковая машина `pending → processing → done | error`: `SEARCH_STATUS_*` (query), `FETCH_STATUS_*` (page); `empty`/`fetched` убраны. Сбой движка → `query_mark_error` (query `error`, **исключение не пробрасывается**); сбой фетча → эта page `error`. **Терминально, без повторов** (nullable колонка `error` — и на query, и на page). Провайдеры `providers/` — **пакеты** (`__init__` регистрирует инстанс через `search_engines.register`/`fetch_engines.register`, `client.py` = **адаптер над коннектором `core_connectors`**: строит params, зовёт gateway, мапит нативный ответ, задаёт `enabled_field=XxxGateway.ENABLED_FIELD`; HTTP/ключ/тумблер — в коннекторе); **два реестра-класса** `SearchEngineRegistry`/`FetchEngineRegistry` (`EngineRegistry[E]`) + синглтоны `search_engines`/`fetch_engines`: `register`/`get(code)`/`codes()` + **учёт доступности через core_connectors** `available_codes()`/`is_available(code)` (по `available()`=`service_enabled(enabled_field)` у каждого движка — `code`/`enabled_field`/`available()` **продублированы** в `SearchEngine`/`FetchEngine`, общего предка нет); отключённый движок поиска → searcher сразу `error` (`error="search_engine_disabled"`); tavily/firecrawl/xai. API `/web-search`: просмотр `GET /queries`(фильтр `search_engine`,`sort_by`)/`/queries/{code}`/`/pages`; запуск `POST /queries` (`CreateQueryBody`→`Searcher.submit`→**202**+`pending` сразу, fire-and-forget; неизвестный движок→400) + `GET /engines` (доступные коды+дефолты для формы). CRUD keyed by code (`query_get/mark_processing/finish/mark_error(code)`, `result_add(query_code=…)`, list order by `created_at`). Настройки (**runtime**, `settings_schema`; фронт `/core/settings`, НЕ ENV): `search_engine`(tavily), `fetch_engine`(tavily), `fetch_concurrency`(5), `default_pages`(10), `max_concurrent_searches`(3). **Троттлинг поисков** (несколько агентов бьют лимиты API): гейт `_acquire_search_slot` перед `query_mark_processing` ждёт (задержка, не ошибка), пока свежих `processing` по движку `< max_concurrent_searches` — **счётчик = строки в БД** (общий для всех процессов, per-provider); reaper **inline** `query_expire_stale` добивает залипшие `processing` >15мин → `error` (планировщика у модуля нет); фетч ограничен транзитивно (слоты×`fetch_concurrency`); rate-гейт (RPM) — отложен. Страница несёт `fetch_engine` (nullable) под per-provider учёт фетча. **Токены** провайдеров тут НЕ хранятся — они в `core_connectors`; `client.py` берёт ключ через `core_connectors.settings.service_api_key`. → [web_search/INDEX](../docs/web_search/INDEX.md)
