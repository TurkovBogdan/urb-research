Once a caveat, prerequisite, or side-note has been delivered and acknowledged, do NOT
re-raise it in subsequent turns of the same thread. State it once, then drop it. The user
got irritated by the cron-recompute prerequisite being repeated at the end of consecutive
answers in the conversation_insights priority discussion.

**Why:** repetition reads as nagging and wastes the user's attention.
**How to apply:** track what's already been said in-thread; reference it tersely ("as noted")
or not at all rather than restating the full point.
