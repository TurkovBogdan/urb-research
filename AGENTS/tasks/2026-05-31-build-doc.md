## Task

User wanted a separate root-level doc holding only project build instructions,
aimed at human contributors ("кожаных"), not agents. Asked first how to name it,
then to create it.

## Context

Build knowledge was spread across `README.md` (mixed with dev workflow) and the
`app_build.py` docstring. No single human-facing build reference at the repo root.

## What was done

- Chose `BUILD.md` (root convention alongside `README.md`, like `INSTALL.md`/`BUILDING.md`).
- Wrote `BUILD.md` sourced from `app_build.py`, `pyproject.toml`, `.env.example`, `README.md`:
  - output layout (`dist/release/` bin + `.desktop` + `.env` + `cert/`), per-OS native build, no cross-compile;
  - requirements table (Python ≥3.11, uv, pnpm);
  - deps (`uv sync --all-groups` + `pnpm --dir web install`), with the `--all-groups` build-group caveat;
  - build command + 3 stages + `app_build.py` flags table (`--clean`/`--skip-web`/`--app`);
  - prod-secrets section (`.env`/`cert/` copied beside the binary, not bundled);
  - running the built binary + icon/`--windowed` notes.

## Result

- Created `BUILD.md` (repo root).
- No code changes; README left as-is (still the dev-workflow entry point).
