## Задача

«Подкрутим порядок настроек: `default_pages` → "Страниц на поисковый запрос", сразу после
сервиса получения контента; две "Максимум параллельных (поисковых запросов / запросов
получения контента)".»

## Что сделано

`settings.py` — новый порядок `SCHEMA` (= порядок рендера на `/core/settings`):
1. `search_provider` «Поисковый движок»
2. `content_provider` «Сервис получения контента»
3. `default_pages` «Страниц на поисковый запрос» (переехал сюда, переименован)
4. `max_concurrent_searches` «Максимум параллельных поисковых запросов» (переименован)
5. `fetch_concurrency` «Максимум параллельных запросов получения контента» (переименован)

Хелперы + `__all__` реордернуты под тот же порядок. Ключи/дефолты/семантика не менялись,
поэтому `searcher.py` не тронут (зовёт настройки по именам, не по лейблам). Docs
(`docs/web_search/INDEX.md` — bullet + таблица) приведены в соответствие.
