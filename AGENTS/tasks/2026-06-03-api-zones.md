## Task

One task, two parts:
1. **Zones** — core owns one zone-router per zone (`src/core/router/internal.py|api.py|webhook.py`),
   modules attach into them instead of `app.include_router("/api/<name>")`. Routes carry a protection
   label (`private` default; `@public` opts out). `internal`=`/internal`, `api`=`/api`, `webhook`=`/webhook`.
2. **Guard passing** — core ships a default guard per label that just **denies** (secure-by-default, no
   auth logic); a registry `set_guard(label, fn)` lets `core_users` replace the `private` (and later
   `signed`/scope) guard with the real session/permission check. The core only routes, reads the label,
   and delegates to whatever guard is registered.

Pattern (user's words): the core `router_internal` collects all modules' routes, labels them `private`,
and its default `private` guard just errors; `core_users` registers a guard/middleware and replaces it.
All auth lives in `core_users` — the core knows nothing about users.

## Context

Today everything is under `/api/<module>` and is in fact SPA-internal (10 module routers + core + settings;
9 frontend `features/*/api.ts` with `BASE='/api/...'`). No external-token API, no webhooks. `api`/`webhook`
are just laid in now (default-deny, no real check). Full design: `AGENTS/docs/api-zones.md`. Pairs with the
error standard [[core_api_standards]] and the auth research [[fastapi-auth]]. `core_users` (models + the
real guard) is a separate later effort that consumes this task's `set_guard`.

## Plan

1. `src/core/router/`: `internal.py`/`api.py`/`webhook.py` (export `router_internal`/`router_api`/`router_webhook`)
   + `labels.py` (`public` marker + `label_of`, default `private`) + `guards.py` (`_deny`, `_GUARDS`,
   `set_guard`, `route_guard`) + `__init__.py`.
2. `create_app`: mount the three zone-routers once with prefixes + `Depends(route_guard)`.
3. Modules attach into the zone-routers; mark public routes with `@public`.
4. Migration `/api/<m>` → `/internal/<m>`: backend mounts, 9 frontend `BASE` (via shared wrapper base
   `/internal`), nginx template (`location /internal/` + `/webhook/`), tests.
5. Infra: `/health` public; OpenAPI tagged by zone.
