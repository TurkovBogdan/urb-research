## Task

> features/settings так-же внедряем — apply the same i18n approach as tasks.

Settings has the same backend-owned-string situation as tasks: each `Field` (in `core._settings`) carries `label`/`description` (and choice options carry `label`), surfaced at `/api/core/settings/modules`. So the same policy applies: the backend sends no key; the frontend derives it from the stable `(module, field key)` identity, with the backend literal as fallback.

## What was done

- **`useSettingText()`** (`features/settings/settingText.ts`) — `localizeField(module, field)` returns a field descriptor with `label`/`description` (and `options[].label`) swapped for translations, all other props preserved. Lookup per leaf: `<module>.setting.<key>.<leaf>` → `settings.catalog.<module>.<key>.<leaf>` (catch-all for modules with no frontend feature dir — currently `llm_providers`) → backend literal. `<leaf>` ∈ {`label`, `description`, `option.<code>`}. Uses `te()` before `t()`.
- **Reactive, no leaf-component changes.** `SettingsView` exposes `localizedModules = computed(() => modules.map(m => ({ ...m, fields: m.fields.map(f => localizeField(m.module, f)) })))`. The 8 `SettingFieldXxx` leaf components keep reading `field.label`/`field.description`/`field.options` — they just receive translated descriptors. Because `localizeField` reads the reactive locale, the computed re-runs on switch → **field labels update live** (verified by clicking the in-page toggle, no reload). `values`/`runtime` keyed by `(module, key)` are untouched.
- **Chrome** (`features/settings/locales/{ru,en}.json`, extending the existing `language` card): `nav`, `page.{title,description}`, `action.{refresh,reset}`, `module.no_fields`, plus the `catalog.llm_providers.*` field translations (5 fields). Sidebar entry got `labelKey: 'settings.nav'` (the entry is an inline object in `AppSidebar.vue`, not a `nav.ts`). Endonyms (`Русский`/`English`) and the module card title (`llm_providers` / the `hh→Headhunter` proper-noun map) kept literal.
- **Shared component:** `SettingFieldList.vue`'s "Добавить" → `t('common.action.add')` — first entries in the previously-empty `web/src/locales/{ru,en}.json` (`common` namespace) for genuinely shared (non-feature) component chrome.

## Result

settings feature fully bilingual, field metadata included. `vue-tsc --noEmit` clean. Verified in-browser ru↔en (reload + **live toggle**): page title/refresh/reset-tooltip switch; all 5 `llm_providers` field labels + descriptions switch (Russian-origin labels translate, English-origin `Temperature`/`max_tokens`/`budget_tokens` stay). Doc `AGENTS/docs/i18n.md` (settings added to migrated list; render-time-computed note). Memory: [[i18n_frontend_setup]].
