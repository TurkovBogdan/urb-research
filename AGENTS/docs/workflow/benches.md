# Бенч-стенды

`dev/bench/` — папка с локальными скриптами для отработки логики поверх **dev-БД**
или реальных API-ответов. Не часть тестовой пирамиды,
не поднимается в CI; запускаются вручную при разработке/отладке.

## Раскладка

```
dev/bench/
    <module>/
        <area>/
            constants.py          # OUT_DIR (tmp/), фикс-id для прогона
            run_<channel>.py      # прогон по подмножеству id, raw + draft в tmp/
            stats_<channel>.py    # покрытие по всей БД, без фильтров
            tmp/                  # артефакты прогонов (.gitignore)
            README.md             # что/как запускать
```

Соглашения:
- **`run_*`** — фокусированные прогоны по фиксированному набору ID / запросов из `constants.py`;
  пишут сырые входы и итоговые выходы в `tmp/`. Папка `tmp/` чистится перед прогоном.
- **`stats_*`** — покрытие/статистика по корпусу. Печатают таблицу результатов.
- Артефакты `tmp/` — рабочий мусор, в git не коммитим (см. `.gitignore`);
  фикстуры для тестов копируются вручную в `tests/.../assets/`.

## Запуск

Из корня репозитория:

```bash
uv run python -m dev.bench.<module>.<area>.run_<channel>
uv run python -m dev.bench.<module>.<area>.stats_<channel>
```

`init_database(Settings())` подхватывает текущий `APP_ENV`
(по умолчанию `dev` → `runtime/dev/`).

## Идея «получить раз, гонять локально»

Получить данные один раз через API, дальше итерировать локально без внешних запросов. Сырые ответы внешних систем кладутся в `raw/`, рядом с каждым — sidecar `*.meta.json` для воспроизведения логики. `<script>.py` помимо `main()` (разовый fetch) экспортирует публичный getter для локальных прогонов без API: getter возвращает уже собранную модель (detached, без commit).

## Когда добавлять новый стенд

- Появилась новая логика поверх большого корпуса данных, нужно мерять покрытие/искать аномалии.
- Логика требует ручного перебора фиксированных кейсов поверх dev-БД или реального API.
- Нужен dataset для последующего создания фикстур теста.

Если задача разовая — обходимся ad-hoc скриптом, не плодим папки.
