# core_setup — страница настроек ENV (правка `.env` + рестарт)

Модуль (`src/modules/core_setup/`) даёт UI/API для редактирования **слоя ENV/`Config`**
(deploy-time: провайдер БД, коннект, порты, воркер). Это НЕ рантайм-настройки
(`core/settings` → `/core/settings`, применяются горячо) — ENV читается на старте,
поэтому изменения применяются **перезапуском процесса**.

**Граница «серверное vs прикладное».** Здесь только то, что нужно, чтобы поднять процесс и
дотянуться до БД: `DB_*` (коннект физически не может жить в БД — курица-яйцо), `SERVER_*`,
`WORKER_*`. Всё прикладное (токены внешних сервисов, выбор провайдера, пороги) — в
runtime-настройках (`/core/settings`, БД, секреты маскируются). Токены web_search жили тут
в группе «Веб-источники» — **переехали** в настройки web_search.

Контекст: локальное приложение, пользователь сам знает, что меняет → **без** auth-гейта,
валидации-с-отказом, бэкапов (сознательно опущены). Маскировки секретов в самом core_setup
тоже нет (например `DB_PASSWORD` виден) — секреты-с-маскировкой живут в runtime-настройках.

## API (internal-зона, `/internal/core/setup`)

- `GET` — поля по группам (Database / Server / Worker) с текущими значениями из `.env`.
- `PUT {values: {KEY: "val"}}` — записать в `.env` → ответ `{status:"restarting"}` →
  отложенный (`0.5s`) `os.execv` перезапуск. Клиент поллит `/internal/health` до возврата.

## Состав

- `keys.py` — `FIELDS`: аллоулист редактируемых ключей (= набор полей формы, не запрет)
  с типом (`str|int|bool|choice`) и `choices`/`secret`.
- `env_file.py` — построчный редактор `.env`: правит `KEY=` **на месте**, комментарии и
  порядок не трогает, недостающие ключи дописывает в конец; читает значения из файла.
  `seed_defaults_if_absent(config)` — **первый запуск без `.env`**: создаёт файл со
  значениями по умолчанию из `Config` (ENV-ключ формы = поле Config в нижнем регистре;
  bool→`true/false`, `None`→пусто). Зовётся из `src/app.py::main` после сборки `Config`,
  до диспетчеризации роли (в тестах не срабатывает — `main()` там не вызывается). Без него
  страница настроек на чистой системе показывала бы пустые поля.
- `restart.py` — `schedule_restart()` → `os.execv(sys.executable, sys.argv)` (без супервизора).
- `api.py` / `module.py` — роутер + `CoreSetupModule` (`internal_router_prefix="/core/setup"`).

## Гранёные углы

- **Рестарт рвёт соединение** (`os.execv`) — фронт переживает поллингом health; при смене
  `SERVER_PORT` UI надо открыть на новом порту.
- **Нет авторизации** — internal-зона под `allow_all`. Для прода — гейт после auth-модуля.
- **Нет валидации** — битое значение (напр. `DB_HOST`) сломает следующий старт; чинить
  правкой `.env` руками. Осознанный trade-off для локального приложения.
